📋 Project Overview

What is PixelNur?

PixelNur is an advanced steganography system that hides secret messages inside images using CNN-based adaptive embedding. The system combines deep learning, wavelet transforms, and cryptography to achieve imperceptible message hiding with high security.

Project Name Origin

"PixelNur" combines:

  • Pixel - The fundamental unit of digital images
  • Nur - Arabic word meaning "light" or "hidden light", symbolizing hidden information

Core Idea & Innovation

Traditional steganography uses simple LSB (Least Significant Bit) replacement, which is easily detectable. PixelNur innovates by:

  • Using a CNN (Convolutional Neural Network) to analyze image texture and identify optimal hiding locations
  • Embedding in the frequency domain using Lifting Wavelet Transform (LWT)
  • Applying adaptive embedding strength based on local image variance
  • Providing attack resistance through Reed-Solomon error correction coding
  • Implementing Sobel gradient fallback when CNN generates invalid masks

🎯 Key Innovation

Unlike traditional methods that embed uniformly across the image, PixelNur uses AI to intelligently select embedding locations based on texture complexity, achieving PSNR >40 dB (visually identical to original) while maintaining high capacity and attack resistance.

🏗️ System Architecture & Data Flow

Complete Embedding Pipeline

1. Input Layer
Inputs: Cover Image (PNG/JPEG) + Secret Message (Text/Binary) + Password (≥16 chars)
File: app.py - Lines 200-250 (embed_interface function)
Validation: Image size (256×256 to 2048×2048), password length, message non-empty
2. CNN Texture Analysis Module
ResNet-18 Backbone + Custom Head
File: src/cnn_module.py - Lines 257-318 (generate_mask method)

Step 2.1: Preprocessing (Lines 188-218)
• Convert BGR → RGB
• Resize to 512×512
• Normalize with ImageNet statistics (mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])
• Convert to PyTorch tensor

Step 2.2: CNN Inference (Lines 105-157 for model, 300-310 for inference)
• Forward pass through ResNet-18 backbone (512 channels, 16×16 resolution)
• Custom head: 512→256→128→64→1 channels with ReLU, BatchNorm, Sigmoid
• Output: Continuous mask values in [0, 1]

Step 2.3: Postprocessing (Lines 219-256)
• Resize mask to LWT sub-band dimensions (H/2 × W/2)
• Apply threshold (default: 0.5) to create binary mask

Step 2.4: Validation & Fallback (Lines 305-318)
• Count non-zero pixels: non_zero_percentage = (count / total) * 100
• If percentage < 5.0% (MASK_VALIDITY_THRESHOLD):
  → Log warning message
  → Call _generate_sobel_fallback_mask() (Lines 280-320)
  → Sobel: Calculate gradients, compute magnitude, threshold at 70th percentile
  → Return fallback mask (~30% non-zero pixels)
• Else: Return CNN mask unchanged
3. Encryption Layer
SHA-256 + XOR Cipher (Slice Encryption)
File: src/encryption_service.py - Lines 50-100

Step 3.1: Key Validation (Lines 30-48)
• Verify password length ≥ 16 characters
• Raise ValueError if invalid

Step 3.2: Hash Generation (Lines 50-60)
• Generate SHA-256 hash: hashlib.sha256(key.encode('utf-8')).digest()
• Result: 32-byte hash digest

Step 3.3: Keystream Generation (Lines 62-75)
• Repeat hash to match message length
• keystream = (hash * repeats)[:message_length]

Step 3.4: XOR Encryption (Lines 77-90)
• encrypted = bytes(a ^ b for a, b in zip(message, keystream))
• Length preservation: len(encrypted) == len(message)
4. Lifting Wavelet Transform (LWT)
2-Level Haar Wavelet Decomposition
File: src/lwt_transform.py - Lines 300-350 (forward method)

Step 4.1: Color Space Conversion (Lines 50-80)
• Convert RGB → YCbCr using ITU-R BT.601 matrix
• Extract Y channel (luminance) for embedding
• Store Cb, Cr channels for later reconstruction

Step 4.2: Level 1 Decomposition (Lines 200-250)
• Apply 2D Haar wavelet transform
• Output: LL1 (approximation), LH1, HL1, HH1 (details)
• Each sub-band: H/2 × W/2 dimensions

Step 4.3: Level 2 Decomposition (Lines 200-250, applied to LL1)
• Apply 2D Haar wavelet to LL1
• Output: LL2, LH2, HL2, HH2
• Each sub-band: H/4 × W/4 dimensions

Result: 7 sub-bands total
• LL2: Low-frequency approximation (NOT used for embedding)
• LH2, HL2, HH2: Level 2 details (HIGH priority for embedding)
• LH1, HL1, HH1: Level 1 details (MEDIUM priority for embedding)
5. Robustness Layer (Optional)
Reed-Solomon Error Correction Coding
File: src/robustness_layer.py

Robustness Levels:
• None (0): No ECC, 100% capacity, no attack resistance
• Low (1): 10 ECC symbols, 80% capacity, light JPEG resistance
• Medium (2): 32 ECC symbols, 27% capacity, moderate attack resistance
• High (3): 64 ECC symbols, 16% capacity, maximum protection

ECC Encoding:
• encoded = rs.encode(message)
• encoded_length = message_length + nsym

Message Replication (Medium/High):
• Replicate message 3× (medium) or 4× (high)
• Majority voting during extraction for error recovery
6. Adaptive Embedding Engine
LSB Matching with Variance-Based Adaptive Strength
File: src/embedding_engine.py - Lines 400-600 (embed method)

Step 6.1: Local Variance Calculation (Lines 150-200)
• Calculate variance in 3×3 neighborhoods
• variance = E[X²] - E[X]² using uniform filters
• Higher variance = more texture = stronger embedding possible

Step 6.2: Adaptive Strength Calculation (Lines 202-250)
• strength = α × w × (1 + log(1 + variance))
• α = base_alpha (default: 0.1)
• w = perceptual weight (LH2/HL2: 1.0, HH2: 0.9, LH1/HL1: 0.7, HH1: 0.6)
• Clamp to [0.5α×w, 2.0α×w] range

Step 6.3: Header Embedding (Lines 450-500)
• Embed 56-bit header in first 56 coefficients of LH2
• Header format: "PNv2" (32 bits) + robustness_level (8 bits) + message_length (16 bits)
• Use fixed strength=3.0 for header robustness

Step 6.4: Priority Queue Creation (Lines 350-400)
• Create max-heap of embedding locations
• Priority = CNN_mask_value × perceptual_weight
• Order: LH2 > HL2 > HH2 > LH1 > HL1 > HH1

Step 6.5: LSB Matching Embedding (Lines 500-600)
• For each message bit:
  → Pop highest priority location from queue
  → Get coefficient and adaptive strength
  → Apply LSB matching: if LSB ≠ bit, modify by ±strength
  → ±1 modification (not direct LSB replacement) for better statistics
